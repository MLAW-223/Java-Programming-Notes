# Methods in Java

## Introduction to Methods

A method in Java is a block of reusable code that is executed when it is called. Think of it like a function or a procedure in other programming languages. Methods are useful because they allow you to write a piece of code once and then reuse it multiple times throughout your program, following the "Don't Repeat Yourself" (DRY) principle.

---

## Defining and Calling Methods

### Method Definition Syntax

To define a method outside of the main method, you use the following structure:

```java
[modifiers] returnType methodName([parameters]) {
    // Method body: code to be executed
    // [return statement;]
}
```

- **modifiers:** Keywords like `public`, `private`, `static`, etc. The video specifically mentions using `static` when calling a method from another static method (like `main`).
- **returnType:** The data type of the value the method will return. If the method does not return any value, use `void`.
- **methodName:** A descriptive name for the method (e.g., `happyBirthday`, `square`, `getFullName`).
- **parameters:** A comma-separated list of variable declarations (data type and name) that the method accepts as input. These are optional.
- **Method Body:** The code block enclosed in curly braces `{}` that contains the instructions the method will execute.
- **return statement:** Used to send a value back from the method to the caller. This is only used if the `returnType` is not `void`.

### Method Call Syntax

To execute a method, you "call" it by its name followed by parentheses `()`:

```java
methodName([arguments]);
```

- **methodName:** The name of the method you want to execute.
- **arguments:** Values or variables passed into the method. These are optional and must match the number, order, and data types of the method's parameters.

---

## Arguments and Parameters

- **Arguments:** The actual values or variables that are passed to a method when it is called. These are provided within the parentheses of the method call.
  - *Example:* In `happyBirthday("Spongebob", 30);`, `"Spongebob"` and `30` are arguments.

- **Parameters:** The variables declared in the method's signature (within the parentheses of the method definition). They act as placeholders for the arguments that will be passed in.
  - *Example:* In `static void happyBirthday(String name, int age)`, `name` and `age` are parameters.

### Key Points:

1. Methods are not familiar with variables declared in other methods. To share data between methods, you must use parameters and arguments.
2. The number, order, and data types of arguments passed during a method call must match the parameters defined in the method signature.
3. Parameter names can be different from argument names, but their data types and order must correspond.

---

## Return Values

Methods can be designed to perform a task and then return a result back to the part of the code that called them.

### `void` Return Type

If a method does not need to return any value, its return type is declared as `void`. The method executes its code, but nothing is sent back to the caller.

```java
static void printMessage() {
    System.out.println("This method doesn't return anything.");
}
```

### Returning a Value

If a method needs to return a value, its `returnType` must be specified (e.g., `int`, `double`, `String`, `boolean`). The `return` keyword is used within the method body to specify the value to be sent back.

**Syntax:**

```java
static dataType methodName(parameters) {
    // ... perform calculations ...
    return value; // 'value' must match 'dataType'
}
```

**How it Works:**

When a method that returns a value is called, the method call itself is replaced by the returned value. This returned value can be:

1. Assigned to a variable.
2. Used directly in an expression or another method call.
3. Printed to the console.

**Example:**

```java
static double square(double number) {
    return number * number; // Returns the calculated square
}

// Calling the method
public static void main(String[] args) {
    double result = square(5.0); // result will be 25.0
    System.out.println(result);  // Output: 25.0

    System.out.println(square(3.0)); // Output: 9.0
}
```

In the `square` example, when `square(5.0)` is called:

1. `5.0` is passed as the `number` argument.
2. The method calculates `5.0 * 5.0`, which is `25.0`.
3. The `return 25.0;` statement sends `25.0` back.
4. The `main` method receives `25.0` and assigns it to the `result` variable.

## Examples from the Video

### `happyBirthday` Method

This method prints a birthday song. It accepts a `name` (`String`) and an `age` (`int`) as parameters. It does not return any value (`void`).

```java
static void happyBirthday(String name, int age) {
    System.out.println("Happy Birthday to you!");
    System.out.printf("Happy Birthday dear %s!\n", name);
    System.out.printf("You are %d years old!\n", age);
    System.out.println("Happy Birthday to you!\n");
}

// Calling it:
// happyBirthday("Spongebob", 30);
```

### `square` Method

This method calculates and returns the square of a given number. It accepts a `double` as a parameter and returns a `double`.

```java
static double square(double number) {
    return number * number;
}

// Calling it:
// double squaredValue = square(4.0); // squaredValue will be 16.0
```

### `cube` Method

This method calculates and returns the cube of a given number. It accepts a `double` as a parameter and returns a `double`.

```java
static double cube(double number) {
    return number * number * number;
}

// Calling it:
// double cubedValue = cube(3.0); // cubedValue will be 27.0
```

### `getFullName` Method

This method concatenates two strings (first name and last name) with a space in between and returns the full name. It accepts two `String` parameters and returns a `String`.

```java
static String getFullName(String first, String last) {
    return first + " " + last;
}

// Calling it:
// String name = getFullName("Patrick", "Star"); // name will be "Patrick Star"
```

### `ageCheck` Method

This method checks if a given age is 18 or older. It accepts an `int` parameter and returns a `boolean` value (`true` if age is 18 or greater, `false` otherwise).

```java
static boolean ageCheck(int age) {
    // Simplified version:
    return age >= 18;

    /*
    // More verbose version:
    if (age >= 18) {
        return true;
    } else {
        return false;
    }
    */
}

// Calling it:
// int userAge = 21;
// if (ageCheck(userAge)) {
//     System.out.println("You may sign up."); // This will be printed
// } else {
//     System.out.println("You must be 18+ to sign up.");
// }
```

## Activity 1: Calculator Toolkit
Create a comprehensive calculator toolkit with methods for different operations:

**Required Methods:**
1. **Basic Operations:**
   - `add(double a, double b)`
   - `subtract(double a, double b)`
   - `multiply(double a, double b)`
   - `divide(double a, double b)` with error handling for division by zero

2. **Advanced Math:**
   - `power(double base, double exponent)`
   - `squareRoot(double number)` with error handling for negative numbers
   - `factorial(int n)` using recursion or loops
   - `gcd(int a, int b)` (Greatest Common Divisor)
   - `lcm(int a, int b)` (Least Common Multiple)

3. **Financial Calculations:**
   - `calculateInterest(double principal, double rate, int time)`
   - `loanPayment(double principal, double rate, int months)`
   - `futureValue(double presentValue, double rate, int periods)`

4. **Unit Conversion Methods:**
   - Temperature: Celsius ↔ Fahrenheit ↔ Kelvin
   - Length: meters ↔ feet ↔ inches
   - Weight: kg ↔ pounds

**Program Requirements:**
- Create a menu-driven interface
- Use method overloading for similar operations
- Implement input validation methods
- Create utility methods for formatting output
- Use helper methods to avoid code duplication

## Activity 2: Game Character System

**Character Methods:**
1. **Creation Methods:**

2. **Combat Methods:**

3. **Leveling Methods:**

**Game Simulation:**
Create a battle simulation that:
- Initializes two characters
- Simulates turn-based combat using the methods
