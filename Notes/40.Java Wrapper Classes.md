# Java Wrapper Classes

## Introduction to Wrapper Classes
Wrapper classes in Java provide a way to treat primitive data types (like `int`, `char`, `double`, `boolean`) as objects. This is achieved by "wrapping" the primitive value within an object.

**Why use Wrapper Classes?**

**Collections Framework:** They are essential for using Java's Collections Framework (e.g., `ArrayList`, `HashMap`), which can only store objects, not primitives directly.

**Object-Oriented Features:** They allow you to leverage object-oriented features and methods associated with objects.

**Utility Methods:** Wrapper classes offer a variety of useful static utility methods for conversions and data manipulation.

Generally, you should only use wrapper classes when you specifically need an object representation of a primitive.

## Autoboxing and Unboxing
**Autoboxing**
Autoboxing is the automatic conversion of a primitive data type to its corresponding wrapper class object.

**Example:**
```java
// Primitive int
int primitiveInt = 123;

// Autoboxing: primitiveInt is automatically wrapped into an Integer object
Integer wrapperInt = primitiveInt;

// Other examples:
Double wrapperDouble = 3.14;
Character wrapperChar = '$';
Boolean wrapperBoolean = true;
```

**Unboxing**
Unboxing is the automatic conversion of a wrapper class object back to its corresponding primitive data type.

**Example:**
```java
// Assume wrapperInt is an Integer object (from autoboxing or explicit creation)
Integer wrapperInt = 123;

// Unboxing: wrapperInt is automatically converted back to a primitive int
int primitiveInt = wrapperInt;

// Other examples:
double primitiveDouble = wrapperDouble;
char primitiveChar = wrapperChar;
boolean primitiveBoolean = wrapperBoolean;
```

## Converting Between Primitives and Strings
Wrapper classes provide static methods to convert between primitive data types and their String representations.

**Converting Primitives to Strings**
The `toString()` static method in each wrapper class can be used to convert a primitive value into a String.

**Syntax:** `WrapperClass.toString(primitiveValue)`

**Examples:**

**Integer to String:**
```java
String strFromInt = Integer.toString(123); // strFromInt will be "123"
```

**Double to String:**
```java
String strFromDouble = Double.toString(3.14); // strFromDouble will be "3.14"
```

**Character to String:**
```java
String strFromChar = Character.toString('$'); // strFromChar will be "$"
```

**Boolean to String:**
```java
String strFromBoolean = Boolean.toString(true); // strFromBoolean will be "true"
```

Concatenating these strings results in a single string:
```java
String combinedString = Integer.toString(123) + Double.toString(3.14) + Character.toString('$') + Boolean.toString(true);
// combinedString will be "1233.14$true"
```

**Converting Strings to Primitives**
The `parseXXX()` static methods in wrapper classes are used to convert String representations into primitive data types.

**Syntax:** `WrapperClass.parseXXX(string)`

**Examples:**

**String to int:**
```java
String intString = "123";
int parsedInt = Integer.parseInt(intString); // parsedInt will be 123
```

**String to double:**
```java
String doubleString = "3.14";
double parsedDouble = Double.parseDouble(doubleString); // parsedDouble will be 3.14
```

**String to boolean:**
```java
String booleanString = "true";
boolean parsedBoolean = Boolean.parseBoolean(booleanString); // parsedBoolean will be true

String falseString = "false";
boolean parsedFalseBoolean = Boolean.parseBoolean(falseString); // parsedFalseBoolean will be false
```

**Note on Characters:** The `Character` wrapper class does not have a `parseChar()` method. To get a character from a string, you typically use the `charAt()` method of the `String` class.

**Example:**
```java
String pizzaString = "pizza";
char firstChar = pizzaString.charAt(0); // firstChar will be 'p'
```

Attempting to add parsed primitives together directly after parsing from strings would result in a compilation error if the types are mixed or if the operation is not defined for the resulting primitive types in that context.

## Miscellaneous Utility Methods
Wrapper classes provide various static utility methods for checking and manipulating primitive values.

**Character Class Utility Methods**
The `Character` wrapper class offers several helpful methods for validating characters.

**`isLetter(char ch)`:** Checks if the given character is a letter.
```java
char letter = 'b';
boolean isLetter = Character.isLetter(letter); // isLetter will be true

char symbol = '$';
boolean isSymbolALetter = Character.isLetter(symbol); // isSymbolALetter will be false
```

**`isUpperCase(char ch)`:** Checks if the given character is an uppercase letter.
```java
char lowerCaseB = 'b';
boolean isBUpperCase = Character.isUpperCase(lowerCaseB); // isBUpperCase will be false

char upperCaseB = 'B';
boolean isUpperCaseB = Character.isUpperCase(upperCaseB); // isUpperCaseB will be true
```

These methods are particularly useful for validating user input, such as usernames or passwords.
