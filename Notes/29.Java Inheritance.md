# Java Inheritance

## What is Inheritance?
Inheritance is a fundamental concept in Object-Oriented Programming (OOP) that allows a class to inherit attributes (variables) and methods (functions) from another class. This promotes code reusability and establishes a hierarchical relationship between classes, similar to how biological inheritance works.

**Parent Class (Superclass/Base Class):** The class that provides its attributes and methods to another class.

**Child Class (Subclass/Derived Class):** The class that inherits attributes and methods from a parent class.

## How to Implement Inheritance
In Java, inheritance is achieved using the `extends` keyword. The child class declaration will include `extends` followed by the name of the parent class.

```java
public class ChildClass extends ParentClass {
    // ... child class members
}
```

## Key Concepts and Examples

### Basic Inheritance
The video introduces a simple inheritance structure with `Animal` as the parent class and `Dog` and `Cat` as child classes.

**Organism Class (Implicitly a grandparent in the multi-level example):**
```java
public class Organism {
    boolean isAlive;

    Organism() {
        isAlive = true;
    }
}
```

**Animal Class (Parent Class):**
```java
public class Animal extends Organism { // Inherits from Organism
    void eat() {
        System.out.println("This animal is eating");
    }
}
```

**Dog Class (Child Class):**
```java
public class Dog extends Animal { // Inherits from Animal
    int lives = 1;

    void speak() {
        System.out.println("The dog goes *woof*");
    }
}
```

**Cat Class (Child Class):**
```java
public class Cat extends Animal { // Inherits from Animal
    int lives = 9;

    void speak() {
        System.out.println("The cat goes *meow*");
    }
}
```

**Example Usage in Main:**
```java
public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog();
        Cat cat = new Cat();

        // Dogs and Cats inherit 'isAlive' from Animal (and indirectly from Organism)
        System.out.println("Is dog alive? " + dog.isAlive); // Output: Is dog alive? true
        System.out.println("Is cat alive? " + cat.isAlive); // Output: Is cat alive? true

        // Dogs and Cats inherit the 'eat()' method from Animal
        dog.eat(); // Output: This animal is eating
        cat.eat(); // Output: This animal is eating

        // Dogs and Cats have their own unique attributes and methods
        System.out.println("Dog lives: " + dog.lives); // Output: Dog lives: 1
        System.out.println("Cat lives: " + cat.lives); // Output: Cat lives: 9

        dog.speak(); // Output: The dog goes *woof*
        cat.speak(); // Output: The cat goes *meow*
    }
}
```

## Benefits of Inheritance
- **Code Reusability:** Avoids duplicating code by defining common attributes and methods in a parent class.
- **Maintainability:** Changes made to the parent class are automatically reflected in all child classes, simplifying updates.
- **Extensibility:** New classes can be created that extend existing ones, adding new functionality without altering the original code.
- **Adherence to DRY Principle:** "Don't Repeat Yourself." Write code once and reuse it.

## Multi-Level Inheritance
Java supports multi-level inheritance, where a class can inherit from another class, which in turn inherits from another class. This creates a chain or hierarchy.

- **Grandparent Class:** The top-most class in a multi-level hierarchy.
- **Parent Class:** Inherits from the grandparent class.
- **Child Class:** Inherits from the parent class.

### Example Structure:
```
Organism (Grandparent)
    |
    v
Animal (Parent, extends Organism)
    |
    +--- Dog (Child, extends Animal)
    |
    +--- Cat (Child, extends Animal)
```

In this structure:
- `Dog` and `Cat` inherit from `Animal`.
- `Animal` inherits from `Organism`.
- Therefore, `Dog` and `Cat` indirectly inherit from `Organism` as well.

**Plant Class Example:**
The `Plant` class is introduced as another child of `Organism`, demonstrating sibling relationships in inheritance.

```java
public class Plant extends Organism { // Inherits from Organism
    void photosynthesize() {
        System.out.println("The plant absorbs sunlight");
    }
}
```

**Example Usage with Multi-Level Inheritance:**
```java
public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog();
        Cat cat = new Cat();
        Plant plant = new Plant();

        // Direct inheritance from Organism
        System.out.println("Dog is alive: " + dog.isAlive); // true
        System.out.println("Cat is alive: " + cat.isAlive); // true
        System.out.println("Plant is alive: " + plant.isAlive); // true

        // Direct inheritance from Animal
        dog.eat(); // This animal is eating
        cat.eat(); // This animal is eating

        // Specific to Plant
        plant.photosynthesize(); // The plant absorbs sunlight

        // Dog/Cat do not inherit methods from Plant (and vice-versa)
        // dog.photosynthesize(); // This would cause a compile-time error
    }
}
```

## Important Notes
- A child class can have its own unique attributes and methods in addition to those inherited from its parent(s).
- A child class can override methods from its parent class to provide a specific implementation.
- Java does not support multiple inheritance for classes (a class cannot directly extend more than one class). However, it does support multiple inheritance through interfaces.
- The `extends` keyword is used to establish the inheritance relationship.

## Types of Inheritance in Java

### 1. Single Inheritance
```java
class A { /* ... */ }
class B extends A { /* ... */ } // B inherits from A
```

### 2. Multi-Level Inheritance
```java
class A { /* ... */ }
class B extends A { /* ... */ } // B inherits from A
class C extends B { /* ... */ } // C inherits from B (and indirectly from A)
```

### 3. Hierarchical Inheritance
```java
class A { /* ... */ }
class B extends A { /* ... */ } // B inherits from A
class C extends A { /* ... */ } // C also inherits from A
```

## Access Modifiers in Inheritance

### Access Levels
| Modifier | Class | Package | Subclass | World |
|----------|-------|---------|----------|-------|
| private | ✓ | ✗ | ✗ | ✗ |
| default (no modifier) | ✓ | ✓ | ✗ | ✗ |
| protected | ✓ | ✓ | ✓ | ✗ |
| public | ✓ | ✓ | ✓ | ✓ |

### Example:
```java
public class Parent {
    private int privateVar;     // Only accessible within Parent
    int defaultVar;             // Accessible within package
    protected int protectedVar; // Accessible within package and subclasses
    public int publicVar;       // Accessible everywhere
}

public class Child extends Parent {
    void accessTest() {
        // privateVar = 10;     // ERROR: private member
        defaultVar = 20;        // OK if in same package
        protectedVar = 30;      // OK
        publicVar = 40;         // OK
    }
}
```

## The `super` Keyword
The `super` keyword is used to refer to the parent class.

### 1. Accessing Parent Class Members
```java
class Parent {
    String name = "Parent";
}

class Child extends Parent {
    String name = "Child";
    
    void displayNames() {
        System.out.println("Child name: " + name);          // Child
        System.out.println("Parent name: " + super.name);   // Parent
    }
}
```

### 2. Calling Parent Class Constructor
```java
class Parent {
    Parent() {
        System.out.println("Parent constructor");
    }
}

class Child extends Parent {
    Child() {
        super(); // Calls parent constructor (implied even if not written)
        System.out.println("Child constructor");
    }
}
```

### 3. Calling Parent Class Methods
```java
class Parent {
    void display() {
        System.out.println("Parent display");
    }
}

class Child extends Parent {
    @Override
    void display() {
        super.display(); // Calls parent's display method
        System.out.println("Child display");
    }
}
```

## Method Overriding
Child classes can override parent class methods to provide specific implementations.

```java
class Animal {
    void makeSound() {
        System.out.println("Animal makes a sound");
    }
}

class Dog extends Animal {
    @Override // Annotation is optional but recommended
    void makeSound() {
        System.out.println("Dog barks");
    }
}

class Cat extends Animal {
    @Override
    void makeSound() {
        System.out.println("Cat meows");
    }
}
```

### Rules for Method Overriding:
1. Must have same method signature (name and parameters)
2. Return type should be same or covariant (subtype)
3. Cannot reduce visibility (e.g., public → private)
4. Cannot override `final` or `static` methods

## The `Object` Class
In Java, all classes implicitly inherit from the `Object` class if they don't explicitly extend another class.

```java
// These two are equivalent
class MyClass { /* ... */ }
class MyClass extends Object { /* ... */ }
```

### Important `Object` class methods:
- `toString()`: Returns string representation of object
- `equals()`: Compares objects for equality
- `hashCode()`: Returns hash code value
- `getClass()`: Returns runtime class of object

## Constructor Chaining in Inheritance
```java
class Grandparent {
    Grandparent() {
        System.out.println("Grandparent constructor");
    }
}

class Parent extends Grandparent {
    Parent() {
        // super() is called implicitly
        System.out.println("Parent constructor");
    }
}

class Child extends Parent {
    Child() {
        // super() is called implicitly
        System.out.println("Child constructor");
    }
}

// Output when creating Child:
// Grandparent constructor
// Parent constructor
// Child constructor
```

## The `final` Keyword in Inheritance

### 1. `final` Class
```java
final class CannotBeExtended { /* ... */ }
// class Child extends CannotBeExtended { } // ERROR: Cannot extend final class
```

### 2. `final` Method
```java
class Parent {
    final void cannotOverride() { /* ... */ }
}

class Child extends Parent {
    // void cannotOverride() { } // ERROR: Cannot override final method
}
```

### 3. `final` Variable
```java
class Parent {
    final int MAX_VALUE = 100; // Cannot be changed
}
```

## Practical Example: Vehicle Hierarchy
```java
// Base class
class Vehicle {
    String brand;
    int year;
    
    Vehicle(String brand, int year) {
        this.brand = brand;
        this.year = year;
    }
    
    void start() {
        System.out.println("Vehicle is starting");
    }
    
    void stop() {
        System.out.println("Vehicle is stopping");
    }
}

// Derived class
class Car extends Vehicle {
    int numberOfDoors;
    
    Car(String brand, int year, int doors) {
        super(brand, year); // Call parent constructor
        this.numberOfDoors = doors;
    }
    
    @Override
    void start() {
        System.out.println("Car engine is starting");
    }
    
    void honk() {
        System.out.println("Car is honking");
    }
}

// Another derived class
class Motorcycle extends Vehicle {
    boolean hasSidecar;
    
    Motorcycle(String brand, int year, boolean hasSidecar) {
        super(brand, year);
        this.hasSidecar = hasSidecar;
    }
    
    @Override
    void start() {
        System.out.println("Motorcycle engine is revving");
    }
    
    void wheelie() {
        System.out.println("Motorcycle is doing a wheelie");
    }
}
```

## Common Mistakes to Avoid

1. **Circular Inheritance:** Class A extends B and B extends A (not allowed)
2. **Forgetting `super()` call** in constructor when parent has no default constructor
3. **Reducing visibility** when overriding methods
4. **Trying to inherit from `final` classes**
5. **Confusing inheritance with composition** (use inheritance for "is-a" relationships)
