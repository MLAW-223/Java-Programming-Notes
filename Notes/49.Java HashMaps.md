# Java HashMaps

## What is a HashMap?
A HashMap is a data structure in Java that stores data as key-value pairs.

**Keys:** Must be unique.

**Values:** Can be duplicated.

**Order:** Does not maintain any specific order of elements.

**Efficiency:** Memory efficient.

## Creating a HashMap
When creating a HashMap, you need to specify the data types for both the key and the value using type parameters. These types must be reference data types (objects, not primitives like `int` or `double`).

```java
import java.util.HashMap; // Import the HashMap class

// HashMap<KeyType, ValueType> variableName = new HashMap<>();
HashMap<String, Double> productPrices = new HashMap<>();
```

In this example:
- `String` is the data type for the keys (e.g., product names).
- `Double` is the data type for the values (e.g., prices).
- The diamond operator `<>` after the second `HashMap` allows Java to infer the type arguments, making the code more concise.

## Adding Elements (Putting Key-Value Pairs)
You use the `put()` method to add elements to a HashMap. This method takes two arguments: the key and its corresponding value.

```java
// map.put(key, value);
productPrices.put("Apple", 0.50);
productPrices.put("Orange", 0.75);
productPrices.put("Banana", 0.25);
```

**Handling Duplicate Keys:** If you use `put()` with a key that already exists, the HashMap will overwrite the previous value associated with that key.

```java
// Example of overwriting a value
productPrices.put("Orange", 1000000.00); // The price of an orange is now updated
```

## Retrieving Elements (Getting Values)
The `get()` method allows you to retrieve the value associated with a specific key.

```java
// map.get(key);
Double applePrice = productPrices.get("Apple"); // applePrice will be 0.50
Double orangePrice = productPrices.get("Orange"); // orangePrice will be 1000000.00 (after the overwrite)
```

## Checking for Keys and Values
**`containsKey(key)`:** Returns `true` if the HashMap contains the specified key, `false` otherwise.

**`containsValue(value)`:** Returns `true` if the HashMap contains the specified value, `false` otherwise.

```java
// Check if a key exists
boolean hasApple = productPrices.containsKey("Apple"); // true
boolean hasPineapple = productPrices.containsKey("Pineapple"); // false

// Check if a value exists
boolean hasPriceOfOneDollar = productPrices.containsValue(1.00); // true (if a key has the value 1.00)
boolean hasPriceOfFiftyCents = productPrices.containsValue(0.50); // false (if the value was overwritten)
```

**Note:** When using `containsValue()`, ensure the data type of the value you are checking matches the value type defined for the HashMap. For example, passing an `Integer` to check for a `Double` value might return `false` even if the numerical value is the same.

## Removing Elements
The `remove(key)` method removes the key-value pair associated with the specified key.

```java
// map.remove(key);
productPrices.remove("Apple"); // Removes the "Apple" key and its value
```

## Getting the Size of the HashMap
The `size()` method returns the number of key-value pairs currently stored in the HashMap.

```java
// map.size();
int numberOfProducts = productPrices.size(); // Returns the current count of items in the map
```

## Iterating Through a HashMap
You can iterate through the key-value pairs in a HashMap using an enhanced for loop. To do this, you typically get a set of all the keys using `keySet()` and then use `get()` to retrieve the value for each key.

```java
// Iterate through keys and print key-value pairs
for (String key : productPrices.keySet()) {
    Double value = productPrices.get(key);
    System.out.println(key + ": $" + value);
}
```

This approach allows for custom formatting of the output, making it easier to read than the default HashMap representation.
